# Правила разработки миниапов (обязательно к прочтению)

> **← [INDEX](INDEX.md)** · Это первый документ. Прочитать до начала работы.

При разработке и деплое миниапов **единственный допустимый канал** — MCP `miniapp` или прямые `/api/` запросы с Bearer-токеном.

---

## Запрещено

- Обращаться к файловой системе сервера напрямую (диски, пути, SSH)
- Подключаться к базе данных любым способом
- Использовать браузерную авторизацию, куки, сессии вне iframe
- Применять любые обходные решения (workaround) при установке или деплое
- Использовать другие MCP-серверы кроме `miniapp`
- Читать или модифицировать серверный код платформы (PHP, SQL, шаблоны)
- Класть в zip серверный код (php, exe, sh и т.д.) — попадут только html/js/css/картинки

## Разрешено

- Писать HTML + JS + CSS + картинки в zip
- Делать `App.fetch('/db/...')` через сессию пользователя
- Делать `/api/...` запросы с Bearer-токеном (для серверных интеграций)
- Создавать каталоги и поля через self-provisioning API
- Использовать `App.storage` для персистентного KV-хранилища приложения
- Использовать удалённое приложение (install_url) вместо zip — тогда серверный код может быть на своей стороне

## Принцип

Если что-то не получается через MCP/API — **остановись и уточни**, не ищи обходной путь.
Любой обход песочницы = уязвимость или привязка к конкретному окружению.
